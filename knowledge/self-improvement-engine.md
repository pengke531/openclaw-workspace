# 自我提升引擎 — 核心机制文档

> AgentOS v2 的核心引擎。确保每次任务都有复盘、每次失败都有归因、每次敷衍都被拦截。

---

## 引擎架构

```
任务完成
  → Post-Task Review（必须）
       ↓
    评分 ≤ 15？
       ↓ yes
    Self-Correction（必须）
       ↓
  经验沉淀 → memory/经验教训.md
       ↓
  每周六 → Weekly Review
       ↓
  系统性问题 → Self-Correction
       ↓
  规则更新 → 相关 doc / skill
```

---

## Post-Task Review — 任务后复盘

### 触发时机
每个任务完成后**立即触发**，不可跳过。

### 填写模板

```markdown
## Post-Task Review — [任务名称]
**日期**: YYYY-MM-DD
**任务分级**: L1 / L2 / L3 / L4
**执行者**: [角色]

### 1. 产出对比
**承诺**: [承诺交付什么]
**实际**: [实际交付什么]
**差距**: [差距分析]

### 2. 敷衍检测
是否有以下敷衍迹象？
- [ ] 文档骨架化
- [ ] 功能无测试验证
- [ ] 用户未确认先 commit
- [ ] "基本完成"搪塞
- [ ] 跳步骤/跳过验证

### 3. 失败分类（如有）
类型: [7+1 类之一]
根因: [为什么会失败]
补救: [如何补救]

### 4. 综合评分 (1-25)
- 产出完整: ___/5
- 流程合规: ___/5
- 用户满意: ___/5
- 经验沉淀: ___/5
- Enforcement 合规: ___/5
**总分**: ___/25

**触发 Self-Correction？**: [是/否]（≤15 是）
```

## Self-Correction — 自我纠正

### 触发条件
- Post-Task Review 评分 ≤ 15
- 同一类失败连续出现 2 次
- Weekly Review 发现系统性敷衍
- 用户明确不满

### 纠正步骤

1. **承认** — 具体承认哪里的问题（不是笼统道歉）
2. **分析** — 找到根本原因（不是表面现象）
3. **纠正** — 立即执行一个具体修复动作
4. **验证** — 确认修复有效
5. **记录** — 把纠正过程写入 `memory/经验教训.md`

### 自我纠正示例

```
问题：README 格式不完整被用户发现
承认：我在 layered-memory-skill 的 README 上写了残缺版本，说好的完整格式是空话
分析：压力下选择走捷径，侥幸心理
纠正：重写 README，按 validator 要求的格式（Overview/Architecture/Usage/Files/License）
验证：运行 validator，ALL CHECKS PASSED
记录：更新 memory/经验教训.md，增加"文档交付前必须自检格式"规则
```

## Enforcement — 强制机制

### Pre-Commit Checklist

在 git commit 前必须逐项确认：

```
Pre-Commit Checklist
[ ] 1. 有实际内容（非骨架文档）
[ ] 2. README 完整（Overview + Architecture + Usage + Files + License）
[ ] 3. 关联文件齐全
[ ] 4. 有测试/验证（非可选）
[ ] 5. 用户已确认（或明确不需要）

通过？[ ] YES → 可以 commit
      [ ] NO（2项以上）→ 禁止 commit
```

### 敷衍行为分类与处理

| 类型 | 触发信号 | 默认处理 |
|------|---------|---------|
| 文档骨架化 | 有标题无内容 | 补充内容或删除 |
| 功能无测试 | 声称完成但无验证 | 补测试或说明验证方式 |
| 用户未确认先 commit | 无确认记录 | 回退，等确认 |
| "基本完成"搪塞 | 使用模糊词 | 补充具体完成状态 |
| 跳步骤 | 跳过验证/确认 | 补全步骤或说明原因 |
| 重复踩坑 | 同类失败第三次 | 触发 Self-Correction |

## 与其他层的关系

- **L1 治理**: 提供任务分级和执行闸门
- **L2 编排**: 提供路由决策（决定任务复杂度）
- **L3 执行**: 提供角色拓扑（决定谁来验证）
- **L4 上下文**: 提供经验存储介质（memory/）
- **L5 工具**: enforcement_validator.py 是 Enforcement 的技术支撑
