# 观测与 Checkpoint 机制

> 让每个复杂任务"可回溯、可分析、可改进"。

---

## 为什么需要 Trace

没有 trace 的任务 = 黑盒操作：
- 成功了不知道为什么会成功
- 失败了不知道哪里出了问题
- 无法从历史中学习，只能重复踩坑

**规则**: L3+ 任务必须产出 trace；L1/L2 任务建议 trace

---

## Trace 字段定义

每个任务的 trace 必须包含：

```markdown
## Trace — [任务名称]

### 基本信息
- 任务 ID:
- 开始时间:
- 结束时间:
- 执行者（角色）:
- 任务分级: L1 / L2 / L3 / L4

### 目标
[最初的目标描述]

### 执行路线
- 步骤 1: [做了什么] → [结果]
- 步骤 2: [做了什么] → [结果]
- 步骤 3: [做了什么] → [结果]

### 最终产物
[实际交付了什么]

### 与目标偏差
[承诺 vs 实际交付的差距]

### 失败记录（如有）
- 失败类型:
- 失败原因:
- 补救措施:

### 关键决策
[执行过程中的关键选择及原因]

### 上下文来源
- L0: [加载了哪些文件]
- L1: [加载了哪些文件]
- L2: [加载了哪些文件]
- L3: [加载了哪些文件]
```

## Checkpoint 时机

### Checkpoint 1: 方案确认后（开始执行前）

**必须确认**:
- 目标是否清晰
- 成功标准是否可验证
- 路线是否合理
- 风险是否已知

**Checkpoint 记录**:
```markdown
## Checkpoint 1 — 方案确认
- 时间:
- 确认结果: [通过 / 不通过]
- 如不通过，原因:
- 调整:
```

### Checkpoint 2: 高风险动作前

**必须确认**:
- 影响范围是否在预期内
- 是否有回滚方案
- 是否获得必要授权
- 是否通知了相关方

**Checkpoint 记录**:
```markdown
## Checkpoint 2 — 高风险动作前
- 时间:
- 高风险动作:
- 影响范围:
- 回滚方案:
- 授权状态:
```

### Checkpoint 3: 阶段性完成后

**必须确认**:
- 当前产物是否满足阶段性标准
- 是否需要调整后续步骤
- 是否需要更新 trace

**Checkpoint 记录**:
```markdown
## Checkpoint 3 — 阶段性完成
- 时间:
- 阶段性产物:
- 阶段性验证结果:
- 后续调整:
```

### Checkpoint 4: 任务完成前（最终验收前）

**必须确认**:
- 所有验收标准是否满足
- 是否有遗漏项
- trace 是否完整
- 是否触发 Post-Task Review

**Checkpoint 记录**:
```markdown
## Checkpoint 4 — 最终验收前
- 时间:
- 验收标准满足情况:
- 遗漏项:
- Post-Task Review 状态: [待触发 / 已完成]
```

---

## 什么任务需要 trace

| 任务等级 | Trace 必需？ | Checkpoint 数量 |
|---------|------------|---------------|
| L1 | 建议（简单记录） | 0 |
| L2 | 建议（简洁记录） | 1（完成后） |
| L3 | 必须 | 2-3 |
| L4 | 必须 | 4（全链路） |

---

## Trace 存储规则

- L3/L4 任务的 trace 必须存储在 `evidence/tool-outputs/`
- 命名格式: `trace-[任务名]-[日期].md`
- 任务完成后，trace 路径必须记录在 Post-Task Review 中
- trace 是经验沉淀的原始材料，必须在 Weekly Review 中被引用

---

## 与其他层的关系

- **L1 治理**: Trace 是 failure taxonomy 的原始数据来源
- **L6 自我提升**: Trace 是 Post-Task Review 和 Weekly Review 的输入
- **L4 上下文**: Trace 存储在 L4 层（原始证据层）

---

## Checkpoint 快速检查清单

```
[ ] 目标是否清晰且可验证？
[ ] 成功标准是否已定义？
[ ] 影响范围是否已知？
[ ] 回滚方案是否已备？
[ ] 高风险操作是否已授权？
[ ] 时间预期是否合理？
[ ] 上下文是否足够且不过载？
```
