# 标准 Handoff 模板

> Agent 间任务交接的标准化格式。所有角色间交接必须使用此模板。

---

## 通用 Handoff 模板

```markdown
## Handoff — [任务名称]
**交接时间**: YYYY-MM-DD HH:MM
**发出角色**: [角色名]
**接收角色**: [角色名]
**任务 ID**: [可选]

### 🎯 目标
[最终要交付什么？一句话描述]

### ✅ 成功标准
- [标准 1：可验证的条件]
- [标准 2：可验证的条件]
- [标准 3：可验证的条件]

### 📋 范围

**做**:
- [具体要做什么]

**不做**:
- [明确什么不在范围内]

### ⚠️ 约束条件
- [硬性限制 1]
- [硬性限制 2]

### 🚫 禁止事项
- [绝对不能做的事 1]
- [绝对不能做的事 2]

### 🔧 可用资源
- [工具 / 文档 / 上下文]

### 📤 输出格式
- [交付物格式描述]
- [文件命名规范]

### ⏰ 时间预期
- 预计完成时间: [时长]
- checkpoint 汇报时间: [时间点]

### 📎 背景与上下文
[必要的背景信息，帮助接收者快速进入状态]
```

## 场景化 Handoff 示例

### 示例 1: 研发类 Handoff（规划 → 实现）

```markdown
## Handoff — 智惠X4 登录模块重构
**交接时间**: 2026-03-23 09:00
**发出角色**: 规划 Agent
**接收角色**: 实现 Agent

### 🎯 目标
将智惠X4 的登录模块从硬编码方式重构为配置化方式

### ✅ 成功标准
- 登录功能正常运行（账号密码 / 短信 / 微信）
- 配置文件可热更新，无需重新部署
- 旧有接口完全兼容，无 Breaking Change
- 有对应的测试用例，覆盖率 >80%

### 📋 范围
**做**: 登录模块重构 + 配置文件设计 + 测试
**不做**: 用户管理模块、数据库结构变更、前端界面修改

### ⚠️ 约束条件
- 使用 Python 3.10+
- 兼容 ARM Windows 环境
- 不得引入新的外部依赖

### 🚫 禁止事项
- 禁止修改数据库表结构
- 禁止改动 API 接口格式
- 禁止删除旧代码（先加新代码，验证后旧代码注释保留）

### 🔧 可用资源
- `项目经验.md` 中智惠X4 相关条目
- `docs/智惠X4_技术开发手册.md`
- 技术栈: Python + FastAPI + MySQL

### 📤 输出格式
- 重构后的登录模块代码
- `config/login_config.yaml` 配置文件
- `tests/test_login.py` 测试文件
- 变更说明文档（300 字以内）

### ⏰ 时间预期
- 预计完成时间: 4 小时
- checkpoint 汇报: 2 小时（中期进度）
```

### 示例 2: 调研类 Handoff（主控 → 调研）

```markdown
## Handoff — AI Agent 框架竞品调研
**交接时间**: 2026-03-23 10:00
**发出角色**: 主控 Agent
**接收角色**: 调研 Agent

### 🎯 目标
调研当前主流 AI Agent 框架（AutoGPT / LangChain / CrewAI / MetaGPT），输出选型建议

### ✅ 成功标准
- 每个框架包含：核心能力、优缺点、适用场景
- 有基于实际使用体验的对比表
- 有针对嬴项目（OpenClaw + 飞书）的选型建议
- 引用来源需可验证

### 📋 范围
**做**: 框架功能调研 + 实际体验记录 + 对比分析
**不做**: 深度源码分析、安装测试、价格谈判条款

### ⚠️ 约束条件
- 优先英文一手文档，中文资料需注明来源
- 每框架至少引用 2 个不同来源
- 选型建议需基于具体场景

### 🚫 禁止事项
- 禁止只引用单一来源
- 禁止只依赖中文资料
- 禁止把 Beta 功能当稳定功能推荐

### 📤 输出格式
- Markdown 格式调研报告
- 含对比表格
- 含参考文献列表
```

### 示例 3: 验证类 Handoff（实现 → 验证）

```markdown
## Handoff — 智惠X4 登录模块验证
**交接时间**: 2026-03-23 14:00
**发出角色**: 实现 Agent
**接收角色**: 验证 Agent

### 🎯 目标
验证登录模块重构是否满足成功标准

### ✅ 成功标准
- [来自原 Handoff 的标准]
- 账号密码登录正常
- 短信登录正常
- 微信登录正常
- 配置文件热更新生效
- API 接口格式不变
- 测试覆盖率 >80%

### 📋 范围
**验证**: 登录模块全部功能 + 配置变更
**不验证**: 用户管理、前端界面

### 🔧 可用资源
- 实现 Agent 提供的代码和配置
- 原 Handoff 中的成功标准
- `docs/智惠X4_技术开发手册.md`

### 🚫 禁止事项
- 禁止自己修改代码
- 禁止跳过任何验收标准
- 禁止给出模糊结论（必须逐项说明）

### 📤 输出格式
```
验证结果:
[逐项列出验证结果：通过 / 不通过 + 具体原因]
综合结论: [通过 / 不通过]
如不通过，需列出:
- 不通过项:
- 修正建议:
```
```

## Handoff 检查清单

发出 Handoff 前必须确认：
- [ ] 目标清晰、可验证
- [ ] 成功标准完整、无歧义
- [ ] 范围明确（做 / 不做都列出）
- [ ] 禁止事项具体
- [ ] 可用资源已提供
- [ ] 输出格式已说明
- [ ] 时间预期合理

收到 Handoff 后必须确认：
- [ ] 目标是否清晰？有疑问立即澄清
- [ ] 成功标准是否可以验证？不能验证的立刻提
- [ ] 是否有越界风险？边界模糊的立刻问
