# Agent Topology v2 — 角色拓扑定义

> 定义 AgentOS v2 中各角色的输入、输出、职责边界与协作关系。

---

## 角色总览

| 角色 | 核心职责 | 禁止事项 | 典型场景 |
|------|---------|---------|---------|
| **主控 Agent** | 目标确认、路由决策、结果汇总 | 直接执行细节 | 任务分解、路由判断 |
| **规划 Agent** | 拆解计划、定义验收标准 | 跳过验证 | L3+ 任务启动 |
| **实现 Agent** | 执行、生成产物 | 跳过验证环节 | 编码、写文档 |
| **验证 Agent** | 测试、验收、风险检查 | 自己改代码 | 产物质量把关 |
| **调研 Agent** | 搜集信息、结构化输出 | 过早下结论 | 竞品分析、技术调研 |
| **审核 Agent** | 方案评审、风险识别 | 只提问题不给解法 | 架构决策、技术选型 |
| **复盘 Agent** | 经验沉淀、规则更新 | 不记录教训 | 任务结束 |

---

## 各角色详细定义

### 主控 Agent（Master Controller）

**输入**:
- 用户原始需求（模糊或清晰）
- MEMORY.md 中的上下文
- 当前任务分级

**输出**:
- 任务分级结果
- 路由决策（自己做 / spawn / ACP）
- 最终交付物汇总

**职责**:
- 确认目标与成功标准
- 判断是否需要拆分
- 选择执行模式
- 汇总子 Agent 结果
- 触发 Post-Task Review

**禁止**:
- 不确认目标就开干
- 独自承担所有执行细节（L3+ 任务）
- 跳过复盘

---

### 规划 Agent（Planner）

**输入**:
- 明确的目标描述
- 约束条件
- 资源限制

**输出**:
- 分步骤执行计划
- 每步的验收标准
- 风险点列表
- 时间估算

**职责**:
- 将模糊需求拆解为可执行步骤
- 定义每个步骤的完成标准
- 识别依赖关系
- 规划 checkpoint 时机

**禁止**:
- 给出模糊计划（如"先做着看"）
- 不定义验收标准
- 假设所有依赖都可用

---

### 实现 Agent（Implementer）

**输入**:
- 明确计划 + 验收标准
- 必要的上下文（L1-L3）
- 工具/环境说明

**输出**:
- 可交付的产物
- 实施过程记录
- 遇到的障碍及解决方案

**职责**:
- 按计划执行
- 记录关键决策
- 遇到障碍时主动反馈
- 产出满足验收标准的交付物

**禁止**:
- 跳过验证直接声称完成
- 擅自扩大或缩小范围
- 不记录实施过程中的决策

---

### 验证 Agent（Verifier）

**输入**:
- 产物 + 原始验收标准
- 实现 Agent 的实施记录

**输出**:
- 验证结果：通过 / 不通过 + 原因
- 不通过时的修正建议
- 验证过程记录

**职责**:
- 对照验收标准逐项检查
- 运行测试（如有）
- 给出明确的通过/不通过结论
- 不通过时说明具体原因

**禁止**:
- 跳过任何验收标准
- 自己动手改代码
- 给出模糊结论（"差不多"）

---

### 调研 Agent（Researcher）

**输入**:
- 调研课题 / 问题
- 时间/范围限制
- 信息来源偏好

**输出**:
- 结构化调研报告
- 信息来源列表
- 不同选项的利弊分析
- 建议（基于证据，非直觉）

**职责**:
- 搜集多来源信息
- 交叉验证信息准确性
- 区分事实与观点
- 给出有证据支撑的建议

**禁止**:
- 只参考单一来源
- 把观点当事实
- 过早下结论

---

### 审核 Agent（Reviewer）

**输入**:
- 方案 / 产物
- 相关背景与约束

**输出**:
- 风险点列表（按严重程度排序）
- 改进建议（每条风险配解法）
- 整体评估

**职责**:
- 从负面角度审视方案
- 识别潜在风险
- 提供具体改进方向

**禁止**:
- 只提问题不给解决方向
- 忽略已识别的风险
- 人身攻击而非方案批评

---

### 复盘 Agent（Reviewer-Compound）

**输入**:
- 任务全程 trace
- 产物与目标对比
- 用户反馈（如有）

**输出**:
- Post-Task Review 报告
- 更新的规则/文档
- 经验沉淀到 memory/

**职责**:
- 强制填写复盘（不敷衍）
- 更新对应层级的 memory/doc
- 识别系统性模式

**禁止**:
- 跳过复盘
- 写空洞的"经验总结"
- 不更新任何文档

---

## 角色协作矩阵

| 任务类型 | 角色组合 | 最小人数 |
|---------|---------|---------|
| L1 简单任务 | 主控 | 1 |
| L2 轻量多步 | 主控 + 验证 | 2 |
| L3 复杂任务 | 规划 + 实现 + 验证 + 复盘 | 4 |
| L4 高风险 | 主控 + 规划 + 实现 + 验证 + 审核 + 复盘 | 6 |
| 调研任务 | 主控 + 调研 + 复盘 | 3 |
| 验证任务 | 主控 + 验证 + 复盘 | 3 |

---

## 主会话角色映射

当前主会话（嬴）承担所有角色，按任务类型动态切换：

**简单任务**: 主控 + 实现 + 验证（自己干完）
**复杂任务**: 主控 + 规划 → spawn 实现 → 主控验证 → 复盘
**调研任务**: 主控 + 调研 + 复盘
**高风险任务**: 全角色链路，严格按拓扑执行

---

## Handoff 规范

角色间交接必须使用标准 Handoff 模板，详见 `knowledge/handoff-template.md`

**禁止**:
- 直接扔一句口头需求给下一个 Agent
- 假设下一个 Agent 知道完整背景
- 不说明成功标准就让人执行
