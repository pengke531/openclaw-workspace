# ACP / 子会话路由策略

> 定义何时使用 OpenClaw ACP / sessions_spawn，何时由主会话直接完成，何时不值得 spawn。

---

## 决策树

```
收到任务
  │
  ├─ L1（直接执行型）─→ 主会话直接完成
  │
  ├─ L2（轻量多步）─→ 主会话完成，但先写简版计划
  │
  ├─ L3（复杂执行）
  │   ├─ 编码/重构 ─→ 必须 ACP sessions_spawn
  │   ├─ 多文件并行 ─→ ACP sessions_spawn
  │   ├─ 调研类 ─→ ACP sessions_spawn（可并行多个子任务）
  │   └─ 单文件改动 ─→ 主会话 + 验证
  │
  └─ L4（高风险）
      └─ → 主会话 + 全链路角色 + 用户确认
```

---

## 必须走 ACP 的场景

### 1. 编码 / 重构类任务
**判断标准**: 需要修改多个文件、涉及逻辑重构、可能有副作用
**原因**: 隔离上下文，避免污染主会话记忆；可独立验证
**模式**: `sessions_spawn` with `runtime="acp"`

### 2. 多文件并行处理
**判断标准**: 可以拆分为相互独立的任务单元
**原因**: 并行加速，隔离失败（一个失败不影响其他）
**模式**: spawn 多个 worker，验证结果后汇总

### 3. 调研类任务（可并行）
**判断标准**: 多个独立调研方向同时进行
**原因**: 节省时间，信息交叉验证
**模式**: spawn 多个调研 Agent，结果汇总到主会话

### 4. 长期任务（>30 分钟）
**判断标准**: 预计总时长超过 30 分钟
**原因**: 避免主会话超时；可中途检查进度
**模式**: ACP sessions_spawn + checkpoint 汇报

### 5. 需要隔离上下文的任务
**判断标准**: 当前上下文与任务无关或可能产生干扰
**原因**: 避免历史上下文污染新任务
**模式**: 新 sessions_spawn，不继承当前上下文

## 主会话直接完成的场景

### 1. 简单问答 / 一步完成
**判断标准**: 回答即可，无需执行动作
**示例**: 查询定义、解释概念、提供建议

### 2. 单文件微小改动
**判断标准**: 改动小于 20 行，不涉及逻辑变更
**示例**: 修文档、改配置、改注释

### 3. 决策 / 判断类任务
**判断标准**: 需要主会话的上下文来做判断
**示例**: 选择方案、评估风险、确认优先级

### 4. 即时性任务
**判断标准**: 需要立即响应，不能等待 spawn 启动
**示例**: 用户正在等待的回复

---

## 不值得 spawn 的场景

### 1. 一步完成的任务
**反例**: "帮我查一下今天的日期"
**原因**: spawn 启动开销大于任务本身

### 2. 总时长 <5 分钟
**反例**: 简单计算、单文件读取
**原因**: 编排成本超过执行成本

### 3. 风险低且无并行收益
**反例**: 整理一份文件列表
**原因**: spawn 不会带来明显收益

### 4. 需要主会话上下文
**反例**: 基于当前讨论的决策
**原因**: 强行 spawn 会丢失上下文

## ACP 使用规范

### sessions_spawn 参数选择

| 参数 | 值 | 适用场景 |
|------|-----|---------|
| `runtime` | `"acp"` | 编码、复杂执行 |
| `runtime` | `"subagent"` | 简单并行任务 |
| `mode` | `"run"` | 一次性任务 |
| `mode` | `"session"` | 需要多轮交互的任务 |
| `cleanup` | `"delete"` | 任务完成后清理 |
| `thread` | `true` | 需要持久化的会话 |

### 传递给子会话的最小信息

**必须包含**:
- 任务目标
- 成功标准
- 约束条件
- 禁止事项
- 输出格式

**禁止**:
- 直接说"帮我做 X"
- 不说明成功标准
- 假设子 Agent 知道完整背景

### 结果回收规范

- 子会话完成后，主会话必须验证结果
- 验证通过才能交付用户
- 验证不通过 → 明确原因 → 决定是回炉还是接受

## 路由决策速查表

| 任务特征 | 路由决策 |
|---------|---------|
| L1 + 一步完成 | 主会话直接做 |
| L2 + 多步 | 主会话 + 简版计划 |
| L3 + 编码 | ACP sessions_spawn |
| L3 + 多文件并行 | ACP sessions_spawn + 多 worker |
| L3 + 调研 | ACP sessions_spawn（并行调研） |
| L3 + 单文件 | 主会话 + 验证 |
| L4 + 高风险 | 主会话 + 全链路 + 用户确认 |
| <5 分钟 | 主会话直接做 |
| 需要主会话上下文 | 主会话直接做 |
| 独立隔离任务 | ACP sessions_spawn |
