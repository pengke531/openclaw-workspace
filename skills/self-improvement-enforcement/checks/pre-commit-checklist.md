# Pre-Commit Checklist — 提交前强制检查

> 本清单在每次 commit 前必须填写。不允许跳过任何一项。

---

## 检查说明

- **通过**: 5/5 项全部满足
- **不通过**: 任何 1 项不满足 → 停止 commit，补全后重检
- **严重不通过**: 任何 3 项以上不满足 → 必须触发 Post-Task 复盘

---

## 清单内容

### ✅ 检查项 1：交付物完整性

**问题**：本次 commit 是否包含规划中的全部核心交付物？

**具体检查**：
- 规划中列出的文件是否全部存在？
- 是否有文件被标记为"稍后补"但实际跳过了？
- 功能实现 vs 功能描述是否一致？

**通过标准**：
- [ ] 规划文件 vs 实际 commit 一致
- [ ] 没有"文件存在但内容残缺"的情况
- [ ] 没有以"简化版"为由跳过核心文件

**如不通过，说明**：_________________________

---

### ✅ 检查项 2：README 完整性

**问题**：README 是否按规范写完整了，不是残缺版？

**具体检查**：
- SKILL.md 的 name 字段与实际目录名是否一致？
- README 是否包含完整说明（不是"稍后补全"）？
- 是否有格式/结构明显不完整的迹象？

**通过标准**：
- [ ] SKILL.md name 字段与目录名一致
- [ ] README 包含：Overview / Architecture / Usage / Files / License
- [ ] 没有明显的"占位内容"

**如不通过，说明**：_________________________

---

### ✅ 检查项 3：关联文件完整性

**问题**：是否有遗漏的关联文件（测试/示例/验证用例）？

**具体检查**：
- 技能类项目：是否有 examples/ 或 tests/？
- 规则类项目：是否有 validation 用例？
- 是否有应该一起 commit 但被遗漏的文件？

**通过标准**：
- [ ] examples/ 目录存在且包含可用示例
- [ ] tests/ 目录存在且包含可运行测试用例
- [ ] 没有遗漏的关联文件

**如不通过，说明**：_________________________

---

### ✅ 检查项 4：真实测试

**问题**：是否经过真实测试，而非仅文件存在检查？

**具体检查**：
- 功能类：是否实际运行并验证了输出？
- 文档类：是否真实展示给用户并得到反馈？
- 规则类：是否有验证用例证明规则有效？

**通过标准**：
- [ ] 功能类：实际运行并记录了输出结果
- [ ] 文档类：至少通过自读验证了内容完整性
- [ ] 有测试结果的记录或截图

**如不通过，说明**：_________________________

---

### ✅ 检查项 5：用户确认

**问题**：是否主动向用户确认交付物符合预期？

**具体检查**：
- 是否向用户展示了本次交付物的实际内容？
- 用户是否有明确的"满意"或"继续"的反馈？
- 是否有用户要求补充但尚未完成的内容？

**通过标准**：
- [ ] 向用户展示了本次所有交付物
- [ ] 用户没有提出未完成的不满
- [ ] 没有"用户还没看就先commit"的情况

**如不通过，说明**：_________________________

---

## 综合判断

| 通过项数 | 判断 | 行动 |
|---------|------|------|
| 5/5 | ✅ 可以提交 | 直接 commit |
| 3-4/5 | ⚠️ 基本合格 | 补充不通过项后再提交 |
| 1-2/5 | 🔴 不合格 | 触发 Post-Task 复盘，补全后重检 |
| 0-1/5 | 🚨 严重敷衍 | 触发自我纠正，禁止本次提交 |
