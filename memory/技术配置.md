# 技术配置

> 用途：L1 技术环境索引层。记录稳定可复用的环境事实、关键约束和默认技术路线。
> 读取规则：排障、环境判断、工具选择前优先读取；需要细节时再下钻到 daily memory 或项目摘要。

## 运行环境基线
- **系统**：Windows ARM64
- **Python**：3.12
- **时区**：Asia/Shanghai
- **代理**：`127.0.0.1:7890`

## OpenClaw / 模型相关
- 本地 Gateway 端口：`127.0.0.1:18789`
- 已知约束：HTTP `POST /v1/chat/completions` 在当前环境下持续读超时，不能作为稳定主链路
- `openclaw agent --agent main` 实测中，`openai-codex` OAuth 刷新会因 `unsupported_country_region_territory` 失败
- 失败后可 fallback 到：`minimax-portal/MiniMax-M2.7`

## 浏览器与账号环境
- Chrome 已登录常用服务，可在需要时通过 `browser` 工具使用

## 语音相关历史约束
- `pyaudio`：编译失败，已放弃
- `edge-tts`：因 `aiohttp` 构建失败不可用
- 曾验证可用：
  - `sounddevice` 录音
  - `pyttsx3` 播报
  - `openWakeWord` + `inference_framework='onnx'`
- 备注：语音项目当前已回滚，不作为现阶段主线

## 默认原则
- 能走 first-class tool 就不走临时野路子
- 不稳定接口不得直接升级为主链路
- 先验证依赖可安装，再设计集成方案
