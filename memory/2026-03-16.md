## Paperclip 集成问题

**问题**：5 个新 agent 在 Paperclip 中显示 error状态

**根本原因**：
- Paperclip 的 Gateway Adapter 发送请求时：agentId="sushi", sessionKey="paperclip"
- Gateway 从 sessionKey 解析 agentId，得到 "paperclip"（不含 agent 信息），fallback 到默认 "main"
- 检查 agentId="sushi" 与解析出的 "main" 不匹配 → 报错

**CEO-嬴政 能工作的原因**：
- Payload 使用 agentId="remote-agent-123"，这是 Gateway 特殊处理的 ID

**解决方案**：
- 临时方案：修改所有 agent 的 Payload 为 {"agentId": "remote-agent-123"}（会失去隔离性）
- 根本方案：修改 OpenClaw Gateway 代码支持多 agent
